高性能 CUDA 加速视频压缩器 - 技术文档
文档版本：1.0
适用场景：极致显卡利用率 + 高画质 + 极速视频压缩（基于 FFmpeg + NVIDIA CUDA）
一、产品定位
本压缩器基于 FFmpeg + NVIDIA NVENC/CUDA 构建，专为极限性能、高压缩比、画质保真设计：

    100% 榨干显卡算力：满负载利用 NVIDIA CUDA 核心 + NVENC 编码单元
    硬件加速：全流程 GPU 解码→预处理→编码，CPU 零瓶颈
    画质优先：可控码率 / 质量模式，兼顾体积与清晰度
    通用兼容：输出 H.265/HEVC、H.264/AVC、AV1（显卡支持）
    命令行一键调用：可集成、可批量、可自动化

    核心优势：比 CPU 压缩快 10~50 倍，比普通 GPU 压缩快 20%~60%，画质不降级。

二、环境依赖
1. 硬件要求

    NVIDIA 独立显卡（支持 NVENC）：GTX 1050Ti+/RTX 20/30/40/50 系列
    显存 ≥ 4GB（4K 视频建议 ≥ 8GB）

2. 软件依赖

    Windows/Linux
    FFmpeg 6.0+ 编译支持 CUDA
    NVIDIA 驱动 ≥ 535.104
    CUDA Toolkit 12.0+（可选，提升性能）

三、核心技术原理
1. 全链路 GPU 流水线
plaintext

输入视频 → CUDA硬件解码 → GPU色彩空间转换 → GPU缩放/预处理 → NVENC硬件编码 → 输出

全程可以不经过 CPU ，零拷贝、极致低延迟、极致速度。（但是可以选使用CPU压缩，同时具有CPU和GPU对比的benchmark功能，调用内置的test.mp4作为样本）
2. 性能压榨策略

    启用 NVENC 高性能模式
    开启 CUDA 多线程并行处理
    锁定最高编码码率控制模式（CQVBR/VBR）
    禁用 CPU 耗时滤镜，全部替换为 CUDA 加速滤镜
    自动匹配显卡最大并行编码帧数量

3. 画质保障策略

    使用 HEVC (H.265) 比 H.264 省 50% 体积
    恒定质量 CQ 模式（人眼无损）
    自适应 B 帧、自适应 GOP、高 Trellis 量化
    保留原始音频轨道（不压缩音质
