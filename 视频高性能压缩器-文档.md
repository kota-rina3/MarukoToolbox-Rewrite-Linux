# 小丸工具箱粉丝致敬版技术文档

## 文档信息

- 文档版本：`1.1.2`
- 适用项目：`小丸工具箱粉丝致敬版`
- 技术栈：`Python + Tkinter + FFmpeg`
- 适用平台：`Windows 10/11`

## 项目定位

小丸工具箱粉丝致敬版是一个面向本地使用的多功能媒体处理工具，目标不是只做单一压缩，而是把视频压缩、音频处理、封装转换、字幕处理、反挤压、LUT 预览、MediaInfo 分析、任务队列和环境检测整合到一个桌面程序里。

它延续了原工具箱“参数集中、批量处理、偏实用”的思路，同时结合 FFmpeg 与现代硬件编码能力，提供更适合当前素材与设备环境的工作流。

## 当前能力概览

### 视频相关

- 支持 CPU / NVIDIA / AMD / Intel QSV 编码器
- 支持 H.264 / H.265 / AV1 / ProRes / VP9 等常见输出方案
- 支持输出倍速，自动同步处理视频时间轴和音频速度
- 支持隐藏水印写入与读取
- 支持导出前二次确认
- 支持编码失败后自动回退 `CPU H.264 / AVC (libx264)` 重试

### 常用工具

- 视频截取
- 分离音视频
- 左右声道复制
- 常用视频素材转音频

### 进阶功能

- AVS 脚本生成、预览、保存、复制
- 反挤压处理
- 字幕封装与导出
- LUT 批量预览
- MediaInfo 参数分析与 A/B 对比
- Benchmark 编码器测试
- 多任务队列、日志窗口、并发处理

## 技术架构

## 界面层

- `main.py`
  - 程序入口。
- `视频高性能压缩器.py`
  - 中文入口脚本，便于直接启动。
- `gui.py`
  - 主界面、页面逻辑、任务管理、设置窗口、预览与交互行为。

## 命令构建层

- `ffmpeg_utils.py`
  - 负责 FFmpeg 命令拼装、预览命令、压缩命令、音频处理命令及部分通用工具函数。

## 配置与数据层

- `config.py`
  - 存放编码器、预设、分辨率、容器、常量配置。
- `data.py`
  - 定义压缩设置、音频设置、环境信息、结果对象等数据结构。

## 核心处理思路

### 1. 图形界面驱动 FFmpeg

程序本质上是一个带状态管理和任务调度能力的 FFmpeg 桌面前端。用户在界面中选择参数后，程序会根据当前页面功能生成对应命令，再通过子进程调用 FFmpeg 执行。

### 2. 按能力自动适配编码器

程序会在环境检测中读取本机 FFmpeg 编码器列表，并结合硬件情况过滤可用选项，避免界面显示用户机器实际不能用的编码器。

### 3. CPU / GPU 混合策略

不同功能不是一味强行走 GPU。

- 纯压缩场景优先利用硬件编码器提升速度。
- 涉及部分滤镜、变速、水印或特殊处理时，会根据命令兼容性切换到更合适的处理路径。
- 当硬件编码失败时，可自动回退到 `libx264` 重新尝试。

### 4. 批处理与任务可视化

所有导出任务都可以直接执行或加入队列。运行期间会输出：

- 主界面状态
- FFmpeg 实时日志
- 单任务进度窗口
- 批量任务完成状态

## 主要页面说明

### 视频页面

负责主压缩流程，包含：

- 编码器选择
- 分辨率与码率控制
- 音频模式
- 额外 FFmpeg 参数
- 隐藏水印
- 输出倍速
- 任务列表与批量导出

当启用“导出前二次确认”时，点击导出或加入任务队列前，会先弹出汇总窗口，显示当前输出关键参数和可能影响画面的设置。

### 音频页面

支持：

- 音频转码
- 从视频提取音频
- 音频倒放

### 常用页面

提供轻量级的常见处理能力：

- 视频截取
- 分离音视频
- 左右声道复制

其中视频截取已针对带 `timecode`、`tmcd`、`data` 附属流的专业素材做兼容处理，默认会排除不适合直接写入目标容器的数据流，降低截取失败概率。

### AVS 页面

支持生成和编辑 AviSynth 脚本，常见能力包括：

- 加黑边
- 裁剪
- 按帧截取
- 亮度调整
- 锐化
- 降噪

脚本预览区域已针对中文显示做优化，减少缺字漏字问题。

### MediaInfo 页面

支持：

- 媒体参数读取
- 关键特性标签高亮
- A/B 对比弹窗
- 差异筛选
- 导出 CSV
- 导出静态帧

### 环境检测与 Benchmark

用于检测：

- FFmpeg / ffprobe 是否可用
- 硬件编码器支持情况
- CPU / GPU 基础信息
- 各编码器样本测试结果

该窗口已优化为相对主窗口居中显示，并尽量保持在主界面前方。

## 设置与界面体验

当前版本包含多项界面可用性优化：

- 支持日间模式 / 黑暗模式切换
- 支持界面文字大小 `小 / 中 / 大` 三档联动
- 主要弹窗默认相对主窗口居中
- 设置未改动时，关闭设置窗口不会再提示保存
- 夜晚模式下修复了勾选文字悬停可见性问题

## 配置与数据存储

程序运行后会在以下目录保存配置和用户数据：

```text
%APPDATA%\marurebuild
```

典型内容包括：

- `app_settings.json`
- 用户预设及其备份
- 运行期配置数据

## 运行依赖

### 基础要求

- Windows 10/11
- Python 3.10+
- FFmpeg / ffprobe 已安装并可调用

### 硬件编码相关

- NVIDIA：需 FFmpeg 支持 `nvenc`
- AMD：需 FFmpeg 支持 `amf`
- Intel：需 FFmpeg 支持 `qsv`

是否可用取决于：

- 当前 FFmpeg 的编译选项
- 显卡驱动是否正常
- 系统环境是否允许对应硬件接口调用

## 已知实现特点

### 导出稳定性

- 支持编码失败后的 CPU H.264 兜底重试
- 如果普通导出和 CPU H.264 兜底都失败，会提示用户是否恢复默认视频压缩设置

### 性能优化

- GPU 占用轮询已转为后台线程
- 消息队列处理改为分批限流
- 可滚动区域刷新加入防抖

这些优化主要用于减少以下操作时的卡顿：

- 拖动窗口
- 滚轮滚动页面
- 切换页面
- 打开部分功能窗口

## 启动方式

```bash
python main.py
```

```bash
python 视频高性能压缩器.py
```

## 说明

- 本项目是桌面媒体工具，不是单一编码脚本集合。
- 不同页面调用的 FFmpeg 策略并不完全相同，这是为了兼顾速度、兼容性和可维护性。
