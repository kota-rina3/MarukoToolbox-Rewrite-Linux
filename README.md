# 小丸工具箱重制版

## 前言
先感谢一下这个工具箱原作者，因为我用小丸工具箱用了有近10年，所以，在其精神之下，出于兴趣重写这个软件，尽量实现原版功能和选项，同时扩展更新。

## 项目简介
小丸工具箱重制版是基于 Python Tkinter + FFmpeg 的本地多功能媒体处理工具，覆盖视频压缩、音频处理、封装转换、反挤压、字幕封装、LUT 预览、MediaInfo 分析、批量任务与 Benchmark。

## 当前版本
- `v1.1.0`

## 主要功能
- 视频压缩：支持 CPU / NVIDIA / AMD / Intel(QSV) 编码器，含常用与高级模式。
- 音频处理：支持音频转码、视频提取音频、倒放、左右声道复制。
- 常用页面：支持视频截取、分离音视频。
- 字幕页面：支持轨道保留/移除、封装、导出字幕。
- 反挤压页面：支持变形素材反挤压、目标画幅与输出分辨率控制。
- 封装页面：支持无损合并、换容器与音频模式切换。
- 复古页面：支持多种老格式转码参数模板。
- LUT 调色页面：支持批量 LUT 缩略图预览。
- MediaInfo：支持关键特性指示灯（杜比、5.1、HDR、10-bit、4K、高帧率、多音轨）。
- 任务管理：支持入队、并发、拖动排序、停止与日志查看。
- Benchmark：支持环境检测、编码器基准测试与快捷勾选（N 卡常用/A 卡常用/I 卡常用/全选/清除）。
- 预设系统：支持“内置预设 + 用户预设”，并可在设置页管理用户预设（导入/导出/删除/排序）。

## 最近已实现的关键改动
- 新增 Intel 核显常用编码器相关支持，并同步到 Benchmark 编码器选择区域。
- 视频页布局优化：输出目录行上移，减少任务列表过长对操作的影响。
- 环境检测与 Benchmark 区域比例优化，修复右侧表格显示不全问题。
- 设置窗口滚动体验优化：支持滚动条、鼠标滚轮、方向键/翻页键滚动，且滚轮不再误切换控件选项。
- 版本号改为手动维护，当前固定 `v1.1.0`。
- 版本显示收敛：仅在程序标题与设置页显示版本。
- Benchmark 新增快捷选择按钮：`N卡常用`、`A卡常用`、`I卡常用`、`全选`、`清除`。
- 主窗口尺寸持久化增强：窗口高度等尺寸可保存；启动时按住 Shift 可触发“是否恢复默认窗口设置”提示。
- 设置页面结构调整：页面排序移到“界面设置”下方，`x264 设置`移动到底部。
- 预设系统升级：视频页区分“用户预设/内置预设”，保存默认进入用户预设；用户预设管理入口统一放入设置页。
- 配置与预设迁移：设置文件、预设备份等统一保存到 `%APPDATA%\\marurebuild`，降低重装丢配置风险。
- 导出记录功能：批量文件超过 10 个时可提示保存 `.log`，记录原文件名、输出目录、开始/结束时间、耗时、输出大小、压缩率。
- 文案与交互优化：`保存设置` 改为 `保存预设`；未选择自定义分辨率时，自定义宽高禁用。
- 常用页新增“分离音视频”和“左右声道复制”能力；音频页支持选择常见视频文件并仅导出音轨。
- MediaInfo 新增特性指示灯（杜比、5.1、HDR 等）并在检测到时高亮。
- 默认行为优化：初始状态文案显示“待命中”；默认页面顺序为“视频、音频、常用”；“变形”改名为“反挤压”。
- 输出目录兜底机制：未配置或路径失效时自动回退到 `D:\\maru-output`，并同步更新界面提示。
- 新增导出文件名格式设置。
- 新增任务结束动作：支持“结束提示音”和“完成后关机”（勾选位于主界面进度条上方）。
- 资源监控区对齐优化：CPU/GPU 文本与图表对齐。
- 全部“输出目录/输出文件夹”按钮统一为淡蓝色样式。

## 配置与数据目录
程序运行后会在以下目录保存配置和用户数据：

```text
%APPDATA%\marurebuild
```

主要包含：
- `app_settings.json`（全局设置）
- 用户预设文件（及备份）
- 运行相关配置文件

## 运行要求
- Windows 10/11
- Python 3.10+
- FFmpeg / ffprobe 可用（已加入 PATH，或放在程序可访问路径）

## 启动方式
```bash
python main.py
```
或：
```bash
python 视频高性能压缩器.py
```

## 项目结构（核心文件）
- `main.py`：入口
- `视频高性能压缩器.py`：中文入口
- `gui.py`：界面与交互逻辑
- `ffmpeg_utils.py`：FFmpeg 命令构建与调用
- `config.py`：配置常量
- `data.py`：数据结构

## 说明
- 不同硬件编码器是否可用，取决于本机 FFmpeg 编译能力与驱动支持。
- 若硬编失败，可切换 `CPU H.264 / AVC (libx264)` 进行兜底。
