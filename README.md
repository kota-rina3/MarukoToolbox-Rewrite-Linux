# 小丸工具箱重制版

## 前言

先感谢一下这个工具箱原作者，因为我用小丸工具箱用了有近10年，所以，在其精神之下，出于兴趣，重写这个软件。尽量实现原版功能和选项，同时扩展更新。

## 这是什么

这是一个基于 Python Tkinter + FFmpeg 的本地视频工具箱，面向经常使用 DaVinci Resolve、Adobe Premiere Pro、剪映专业版等软件的创作者和后期工作流。它把常用的视频压缩、代理文件、封装转换、LUT 预览、音频转换、MediaInfo 信息查看和批量任务管理集中到一个桌面 GUI 中。

## 功能特性

- 视频压缩：支持 H.265 / H.264 / AV1 等常用编码格式。
- 硬件编码：支持 NVIDIA NVENC、AMD AMF，也保留 CPU x264 / x265 编码路径。
- 批量处理：可批量扫描文件夹，支持保留目录结构。
- 代理文件：一键生成剪辑用低码率代理文件。
- 任务队列：可将视频压缩任务加入队列，支持并发数量和拖拽排序。
- CPU/GPU 监控：顶部显示 CPU/GPU 占用率，并提供简易绿色趋势图。
- LUT 预览：批量生成 LUT 缩略图，横向排列便于对比。
- 音频处理：支持 AAC、MP3、Opus、FLAC、WAV 等格式转换。
- 复古格式：支持 AVI、WMV、ASF、RM、MPG、FLV 等老格式输出。
- 封装转换：可复制视频流，只更换容器或将音频转为 AAC。
- MediaInfo：通过 ffprobe 读取媒体文件的封装、视频流、音频流信息。
- 设置管理：支持保存/导入压缩设置、x264 设置、自定义预览播放器、托盘模式等。

## 运行环境

使用前提：电脑上需要已经安装完整 FFmpeg 环境，并且 `ffmpeg` / `ffprobe` 可以在命令行中直接调用。如果没有安装，需要先自行前往 FFmpeg 官方网站或可信发行源下载并配置 PATH。

建议环境：

- Windows 10 / Windows 11
- Python 3.10+
- FFmpeg 6.0+
- ffprobe

可选依赖：

- `tkinterdnd2`：用于增强文件拖拽导入体验。如需安装，可运行 `pip install tkinterdnd2`。

硬件编码支持取决于本机 FFmpeg 编译版本和显卡驱动：

- NVIDIA：需要 FFmpeg 支持 `hevc_nvenc`、`h264_nvenc`、`av1_nvenc`
- AMD：需要 FFmpeg 支持 `hevc_amf`、`h264_amf`、`av1_amf`
- CPU：需要 FFmpeg 支持 `libx264`、`libx265` 等编码器

## 安装与启动

1. 安装 Python。
2. 安装 FFmpeg，并将 `ffmpeg.exe` 和 `ffprobe.exe` 加入系统 PATH。
3. 克隆或下载本项目。
4. 在项目目录运行：

```bash
python main.py
```

也可以运行中文入口：

```bash
python 视频高性能压缩器.py
```

## 基本使用

### 视频压缩

1. 在“视频”页面添加视频文件或文件夹。
2. 选择编码器、质量、音频模式、输出容器和分辨率。
3. 选择文件冲突策略：跳过、覆盖、重命名。
4. 点击“开始压缩”。

需要更多参数时，勾选“高级模式”，可显示自定义命令、目标码率、LUT、缩略图时间等进阶选项。

### 任务队列

“视频任务入队”表示把当前“视频”页面里的视频压缩任务加入“任务管理”页面。加入队列时会保存当前的视频列表、输出目录和压缩参数快照。

在“任务管理”页面可以：

- 启动选中任务
- 启动全部任务
- 设置同时进行任务数
- 拖拽任务行调整顺序
- 移除未运行任务
- 清除已完成任务

### 批量压缩

“批量压缩”页面适合整盘素材归档。选择输入/输出文件夹和批量预设后开始扫描处理。

内置预设包括：

- H.265 4K
- H.265 2K
- H.265 1080p
- H.265 720p
- H.264 1080p

可勾选“竖屏模式”，将横屏分辨率映射为对应竖屏规格。

### 代理文件

在“视频”页面点击“生成代理文件”，会输出适合剪辑预览的轻量 MP4 代理文件。

### LUT 预览

在“Lut调色”页面选择视频和 LUT 文件夹，点击生成后，下方会横向显示每个 LUT 的预览缩略图。鼠标悬停 1 秒可查看 LUT 文件名。

### MediaInfo

在“MediaInfo”页面打开或拖入媒体文件，可查看：

- 文件名和封装格式
- 文件体积、时长、整体码率
- 视频编码、分辨率、帧率、色彩信息
- 音频编码、码率、采样率、声道

## 设置说明

设置窗口包含：

- 界面设置：语言、托盘模式
- x264 设置：优先级、线程数量、自定义命令行
- 功能设置：预览播放器、拖拽导入、覆盖提醒、任务完成后打开目录
- 编码器偏好：优先 GPU / 优先 CPU
- 日志维护：查看日志、删除日志
- 默认恢复：还原默认设置

应用设置会保存到：

```text
data/app_settings.json
```

## 项目结构

```text
.
├── main.py                  # 主入口
├── 视频高性能压缩器.py       # 中文入口
├── gui.py                   # Tkinter GUI 和任务逻辑
├── ffmpeg_utils.py          # FFmpeg / ffprobe 命令构建和媒体信息读取
├── config.py                # 配置、预设、编码器列表
├── data.py                  # 数据结构
├── data/                    # 应用设置目录
├── Lut/                     # LUT 示例文件
└── output/                  # 输出目录
```

## GitHub 提交建议

不建议提交以下文件或目录：

- `output/`
- `__pycache__/`
- `*.log`
- 大体积测试视频，例如 `test.MP4`
- 临时 FFmpeg pass log，例如 `ffmpeg2pass-0.log`

建议添加 `.gitignore` 后再上传仓库。

## 注意事项

- 本项目依赖本机 FFmpeg 能力，某些编码器是否可用取决于 FFmpeg 编译参数和显卡驱动。
- AMD AMF、NVIDIA NVENC、AV1 编码并非所有显卡都支持。
- RM、WMV、ASF 等老格式输出能力取决于本机 FFmpeg 是否启用对应 muxer/encoder。
- 托盘模式当前表现为点击关闭按钮后隐藏主窗口，任务仍在后台运行。
